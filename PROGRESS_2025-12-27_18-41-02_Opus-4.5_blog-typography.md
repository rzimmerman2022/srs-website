## 3. Technical Decisions Made

### Decision 1: Abandon Prose Classes for Custom CSS
**Rationale:** Tailwind's prose-* classes weren't working because @tailwindcss/typography plugin wasn't installed. Even after installing, explicit CSS gave more control.
**Trade-offs:** More CSS code, but deterministic results
**Alternatives Rejected:**
- Fix prose classes (still dependent on plugin behavior)
- Inline styles (not maintainable)

### Decision 2: 680px Max Width for Blog Content
**Rationale:** Gold standard research shows 50-75 characters per line optimal for readability. 680px achieves this with 18px font.
**Trade-offs:** Narrower content area, but significantly improved readability
**Alternatives Rejected:**
- max-w-4xl (896px) - too wide, causes eye strain
- max-w-xl (576px) - too narrow for comfortable reading

### Decision 3: Gold (#d4af37) for List Markers
**Rationale:** Brand color consistency, creates visual hierarchy, distinguishes lists from body text
**Trade-offs:** Non-traditional bullet color
**Alternatives Rejected:**
- Default black bullets - generic
- Navy bullets - not enough contrast

## 6. Knowledge Gained

### Insights About the Codebase
1. **Typography plugin was missing** - All prose-* classes were essentially no-ops
2. **Blog content uses dangerouslySetInnerHTML** - HTML must be well-structured in posts.ts
3. **Design system uses gold (#d4af37) as accent** - Should be used consistently
4. **Case studies on /results page** - Full content exists, just needs anchor navigation

### Gotchas Discovered
1. Tailwind's prose classes require @tailwindcss/typography plugin - not included by default
2. List bullets need explicit `list-style-type: disc` - Tailwind resets remove defaults
3. CSS `::marker` pseudo-element only works for list items, not arbitrary elements

### Patterns Identified
1. **Blog posts follow consistent structure:**
   - `<p class="lead">` for opening paragraph
   - `<div class="callout-box">` for key takeaways
   - `<div class="cta-box">` for calls to action
   - `<h2>` for main sections, `<h3>` for subsections
